DEMO 

1st Graph

This shows just the Pareto front - the 7 points that represent the best trade-offs. The 2D algorithm only identifies Front 0, so we're seeing those optimal points connected by the red line.

These points form the boundary - if I had shown all 200 points, everything else would be 'behind' this line, dominated by at least one point on the front.

The 2D algorithm is optimized to find just the Pareto front quickly. It doesn't compute the secondary fronts. The n-D algorithm does return all fronts, so you'd see the gray dominated points there. Adding full front ranking to the 2D case is a possible enhancement.
